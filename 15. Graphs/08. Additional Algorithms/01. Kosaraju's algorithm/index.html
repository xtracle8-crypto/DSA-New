Problem Statement

You are given a directed graph with V vertices, numbered from 0 to V − 1, and its adjacency list Adj, where Adj[i] contains all vertices j such that there is a directed edge from vertex i to vertex j.

Your task is to find the number of strongly connected components (SCCs) in the graph.

Constraints

1 ≤ V ≤ 5000

0 ≤ E ≤ (V*(V-1))

0 ≤ ai, bi ≤ V-1

Fun Fact

Fun Fact: The concept of finding strongly connected components in a directed graph has real-world applications in many fields like social media, search engines, and recommendation systems. Did you know, in social media, it is used to identify tightly-knit community structures? For example, one where everyone is friends with everyone else or everyone follows everyone else. It also helps in analyzing web pages, where each page is a vertex and hyperlinks are directed edges. By finding the strongly connected components, search engines can identify groups of pages that are highly interlinked, improving search results. Pre Requisite: DFS traversal technique

What is meant by Strongly Connected Component (SCC)?

A component is called a Strongly Connected Component(SCC) only if for every possible pair of vertices (u, v) inside that component, u is reachable from v and v is reachable from u.
Note: Strongly connected components(SCC) are only valid for directed graphs.

By definition, within each SCC, every node is reachable. So, DFS traversal is started from a node of SCC1, all the nodes in SCC1 can be visited and via edge e1 the SCC2 can be reached. Now, in this graph, if a DFS is started from node 0, it will only visit nodes in SSC1. It is true for all other nodes in Strongly Connected Components. Hence, it is clear that in one DFS call, all the nodes in one SCC will be visited. Thus, the number of DFS calls will represent the number of strongly connected components.

How to identify the edges that must be reversed?
To find the edges that must be reversed, the strongly connected components must be known prior. To solve this problem, all the edges in the graph can be reversed, this will not affect the strongly connected components as already all nodes are reachable to each other. How to identify the starting node for DFS traversal?
Consider if instead of node 0, node 7 is used as the starting node for DFS traversal. In such a case, even after reversing the edges, all the nodes will be visited. Thus, it is important to identify the node from where the DFS can be started so that the SCCs can be identified.

This can be solved by identifying the order of visiting the nodes when a DFS traversal is made on the original graph. The node that will be visited later will have a greater finishing time, and the node that will be visited before will have a smaller finishing time.

Using this, the order in which the nodes must be visited to determine the number of SCCs can be determined based on their finishing time. The DFS calls for the node that was visited first(having the smallest finishing time) must be made first and for the node that was visited last(having the greatest finishing time) must be made last.

Note: Since the node visited last in the original DFS call must be visited first in the later DFS call, a Stack data structure can be used to store the nodes based on their finishing times.

Kosaraju's Algorithm:

  • Sort all the nodes according to their finishing time: Perform a DFS call to sort the nodes based on their finishing time and store them in a stack.
  • Reverse all the edges of the entire graph: Create a new graph where all the edges of the original graph will be reversed.
  • Perform the DFS and count the number of different DFS calls to get the number of SCC: Start DFS traversal on the reversed graph from the node which is on the top of the stack and continue until the stack becomes empty. For each DFS call, the counter representing the number of SCCs can be incremented by 1.

Approach:

  1. Create a visited array to keep track of visited nodes and a stack to store nodes based on their finishing times during DFS traversal.
  2. Perform a DFS on the original graph to determine the finishing times of nodes. Each node is pushed onto the stack after all its descendants are processed.
  3. Reverse the edges of the original graph to create a transposed graph. Perform DFS on the transposed graph, starting from the nodes in the order defined by the stack (based on their finishing times from the first DFS).
  4. Count the number of DFS trees formed in this step, which corresponds to the number of strongly connected components.
  5. The count of DFS trees from the second pass is the number of strongly connected components in the original graph.

    void dfs(int node, vector &vis, vector adj[],

        // Visited array
        vector vis(V, 0);
        
        // Stack data structure 
        stack st;
        
        /* Perform initial DFS to store the nodes 
        in stack based on their finishing time */
        for (int i = 0; i < V; i++) {
            if (!vis[i]) {
                dfs(i, vis, adj, st);
            }
        }
        
        // To store the revered graph
        vector adjT[V];
        
        /* Reverse all the edges of original 
        graph to the reversed graph */
        for (int i = 0; i < V; i++) {
            
            // Mark the node as unvisited
            vis[i] = 0;
            
            // Add the reversed edge
            for (auto it : adj[i]) {
                adjT[it].push_back(i);
            }

        /* To store the count of strongly 
        connected components */
        int count = 0;

        node based on their finishing time */
        while (!st.empty()) {
            // Get the node
            int node = st.top();
            st.pop();
            
            // If not visited already
            if (!vis[node]) {
                count += 1;
                helperDFS(node, vis, adjT);
            }

        return count;

package main

import "fmt"

// dfs performs DFS and stores nodes in the stack based on their finishing time
func dfs(node int, vis []bool, adj [][]int, st *[]int) {
	// Mark the node as visited
	vis[node] = true

	// Traverse all its neighbors
	for _, it := range adj[node] {
		if !vis[it] {
			dfs(it, vis, adj, st)
		}
	}

	// Push the node in stack
	*st = append(*st, node)
}

// helperDFS performs DFS on the reversed graph to find SCCs
func helperDFS(node int, vis []bool, adjT [][]int) {
	// Mark the node as visited
	vis[node] = true

	// Traverse all its neighbors
	for _, it := range adjT[node] {
		if !vis[it] {
			helperDFS(it, vis, adjT)
		}
	}
}

// kosaraju finds the number of strongly connected components
func kosaraju(V int, adj [][]int) int {
	// Visited array
	vis := make([]bool, V)

	// Stack data structure
	st := []int{}

	// Perform initial DFS to store the nodes in stack based on their finishing time
	for i := 0; i < V; i++ {
		if !vis[i] {
			dfs(i, vis, adj, &st)
		}
	}

	// Reverse all the edges of the original graph
	adjT := make([][]int, V)
	for i := 0; i < V; i++ {
		vis[i] = false // Mark unvisited again
		for _, it := range adj[i] {
			adjT[it] = append(adjT[it], i)
		}
	}

	// Count of strongly connected components
	count := 0

	// Process nodes in the order of finishing time
	for len(st) > 0 {
		// Pop the top element
		node := st[len(st)-1]
		st = st[:len(st)-1]

		if !vis[node] {
			count++
			helperDFS(node, vis, adjT)
		}
	}

	// Return the result
	return count
}

	V := 5
	edges := [][]int{
		{1, 0}, {0, 2},

	adj := make([][]int, V)

Complexity Analysis:

Time Complexity: O(V+E) (where E represent the number of edges)
Two DFS traversals are made each taking an O(V+E) time. Reversing all the edges in the graph takes O(E) time.

Space Complexity: O(V+E)
Storing the transposed graph takes O(V+E) space and the space due to the stack and visited array will be O(V) each.

{
    /* Function to perform DFS for storing the nodes 
    in stack based on their finishing time */
    private void Dfs(int node, bool[] vis, List<List<int>> adj, Stack<int> st)
    {
        // Mark the node as visited
        vis[node] = true;

        // Traverse all its neighbors
        foreach (int neighbor in adj[node])
        {
            if (!vis[neighbor])
            {
                // Recursively perform DFS if not visited already
                Dfs(neighbor, vis, adj, st);
            }
        }

        // Push the node in stack
        st.Push(node);
    }

    /* Helper function to perform DFS for finding 
    number of Strongly connected components */
    private void HelperDfs(int node, bool[] vis, List<List<int>> adjT)
    {
        // Mark the node as visited
        vis[node] = true;

        // Traverse all its neighbors
        foreach (int neighbor in adjT[node])
        {
            if (!vis[neighbor])
            {
                // Recursively perform DFS if not already visited
                HelperDfs(neighbor, vis, adjT);
            }
        }
    }

    /* Function call to find the number of 
    strongly connected components */
    public int Kosaraju(int V, List<List<int>> adj)
    {
        // Visited array
        bool[] vis = new bool[V];

        // Stack data structure 
        Stack<int> st = new Stack<int>();

        /* Perform initial DFS to store the nodes 
        in stack based on their finishing time */
        for (int i = 0; i < V; i++)
        {
            if (!vis[i])
            {
                Dfs(i, vis, adj, st);
            }
        }

        // To store the revered graph
        List<List<int>> adjT = new List<List<int>>();
        for (int i = 0; i < V; i++)
        {
            adjT.Add(new List<int>());
        }

        /* Reverse all the edges of original 
        graph to the reversed graph */
        for (int i = 0; i < V; i++)
        {
            // Mark the node as unvisited
            vis[i] = false;

            // Add the reversed edge
            foreach (int neighbor in adj[i])
            {
                adjT[neighbor].Add(i);
            }
        }

        /* To store the count of strongly 
        connected components */
        int count = 0;

        /* Start DFS call from every unvisited 
        node based on their finishing time */
        while (st.Count > 0)
        {
            // Get the node
            int node = st.Pop();

            // If not visited already
            if (!vis[node])
            {
                count += 1;
                HelperDfs(node, vis, adjT);
            }
        }

        // Return the result
        return count;
    }
}

// dfs performs DFS and stores nodes in the stack based on their finishing time
func dfs(node int, vis []bool, adj [][]int, st *[]int) {
	// Mark the node as visited
	vis[node] = true

	// Traverse all its neighbors
	for _, it := range adj[node] {
		if !vis[it] {
			dfs(it, vis, adj, st)
		}
	}

	// Push the node in stack
	*st = append(*st, node)
}

// helperDFS performs DFS on the reversed graph to find SCCs
func helperDFS(node int, vis []bool, adjT [][]int) {
	// Mark the node as visited
	vis[node] = true

	// Traverse all its neighbors
	for _, it := range adjT[node] {
		if !vis[it] {
			helperDFS(it, vis, adjT)
		}
	}
}

// kosaraju finds the number of strongly connected components
func kosaraju(V int, adj [][]int) int {
	// Visited array
	vis := make([]bool, V)

	// Stack data structure
	st := []int{}

	// Perform initial DFS to store the nodes in stack based on their finishing time
	for i := 0; i < V; i++ {
		if !vis[i] {
			dfs(i, vis, adj, &st)
		}
	}

	// Reverse all the edges of the original graph
	adjT := make([][]int, V)
	for i := 0; i < V; i++ {
		vis[i] = false // Mark unvisited again
		for _, it := range adj[i] {
			adjT[it] = append(adjT[it], i)
		}
	}

	// Count of strongly connected components
	count := 0

	// Process nodes in the order of finishing time
	for len(st) > 0 {
		// Pop the top element
		node := st[len(st)-1]
		st = st[:len(st)-1]

		if !vis[node] {
			count++
			helperDFS(node, vis, adjT)
		}
	}

	// Return the result
	return count
}

        # Traverse all its neighbors

            if not vis[it]:
                # Recursively perform DFS if not visited already
                self.dfs(it, vis, adj, st)
        
        # Push the node in stack

    def helperDFS(self, node, vis, adjT):

        vis[node] = 1
        
        # Traverse all its neighbors
        for it in adjT[node]:
            if not vis[it]:
                # Recursively perform DFS if not already visited
                self.helperDFS(it, vis, adjT)
    
    def kosaraju(self, V, adj):
        # Visited array

        # Stack data structure

        ''' Perform initial DFS to store the nodes 
        in stack based on their finishing time '''
        for i in range(V):
            if not vis[i]:
                self.dfs(i, vis, adj, st)
        
        # To store the reversed graph
        adjT = [[] for _ in range(V)]
        
        ''' Reverse all the edges of original 
        graph to the reversed graph '''
        for i in range(V):
            # Mark the node as unvisited
            vis[i] = 0
            
            # Add the reversed edge
            for it in adj[i]:
                adjT[it].append(i)
        
        ''' To store the count of strongly 
        connected components '''
        count = 0
        
        ''' Start DFS call from every unvisited 
        node based on their finishing time '''
        while st:
            # Get the node
            node = st.pop()
            
            # If not visited already
            if not vis[node]:
                count += 1
                self.helperDFS(node, vis, adjT)
        
        # Return the result
