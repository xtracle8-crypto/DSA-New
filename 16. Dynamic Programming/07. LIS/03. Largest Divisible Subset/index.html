Largest Divisible Subset

Medium dsa
ArrayDynamic ProgrammingMathSorting

Problem Statement

Given an array nums of positive integers, the task is to find the largest subset such that every pair (a, b) of elements in the subset satisfies a % b == 0 or b % a == 0.


Return the subset in any order. If there are multiple solutions, return any one of them.


Note: As there can be multiple correct answers, the compiler returns 1 if the answer is valid, else 0.

Constraints

  • 1 <= nums.length <= 103
  • 1 <= nums[i] <= 106

Example 1

Input: nums = [3, 5, 10, 20]

Output: [5, 10, 20]

Explanation:

The subset [5, 10, 20] satisfies the divisibility condition: 10 % 5 == 0 and 20 % 10 == 0.

Example 2

Input: nums = [16, 8, 2, 4, 32]

Output: [2, 4, 8, 16, 32]

Explanation:

The entire array forms a divisible subset since 32 % 16 == 0, 16 % 8 == 0, and so on.

Example 3

Input: nums = [7, 14, 28, 3]

Editorial: Solution

Pre-Requisite: Longest Increasing Subsequence and Print LIS.

Intuition

The crucial insight is that if we sort the array first, the divisibility condition becomes much simpler to handle.

Consider three numbers: a < b < c. If a divides b and b divides c, then a must also divide c (transitivity of divisibility). This means:

  • In a valid divisible subset, if we sort the elements, each element must divide all elements that come after it
  • We can transform this into a "chain" problem where we only need to check if the current element divides the next element

After sorting, our problem transforms from:
  • From: Finding subset where every pair is divisible.
  • To: Find longest chain where each element divides the next.

This is exactly the LIS pattern! Instead of checking arr[i] < arr[j], we check arr[j] % arr[i] == 0.

Approach

  • Sort the Array: Sort input in ascending order to transform the problem into finding the longest divisible chain.
  • Initialize Arrays:
    • dp[i] = length of longest divisible subset ending at index i (initially 1)
    • parent[i] = previous element in optimal subset (initially i)
  • Fill DP Table: For each element, check all previous elements. If divisible and gives longer subset, update dp[i] and parent[i].
  • Track Maximum: Keep track of maximum length and its ending index (lastIndex) while filling DP table.
  • Reconstruct Subset: Backtrack from lastIndex using parent array until reaching an element that is its own parent.
  • Return Result: The backtracked elements form the longest divisible subset in correct order.

using namespace std;

class Solution {
public:
    // Function to find the largest divisble subset
    vector largestDivisibleSubset(vector nums) {
        int n = nums.size(); // Size of the array 

        sort(nums.begin(), nums.end());

        vector ans; // To store the LDS
        vector dp(n, 1); // DP array 
        vector parent(n); // Array to keep record of the parent

        // To store the index of last element in the LDS
        int lastIndex = 0; 

        // To store the length of LDS
        int maxLen = 0;

        // Computing the DP array 
        for(int i = 0; i < n; i++) {
            parent[i] = i; // Assign the parent to itself

            // For each previous index
            for(int prevInd = 0; prevInd < i; prevInd++) {
                
                // If the element at index i can be included in the LDS ending at index j
                if(nums[i] % nums[prevInd] == 0 && dp[i] < dp[prevInd] + 1) {

                    dp[i] = dp[prevInd] + 1; // Update the DP value
                    parent[i] = prevInd; // Store the parent
                }
            }

            // If a longer LDS is found, update the values
            if(dp[i] > maxLen) {
                lastIndex = i;
                maxLen = dp[i];
            }
        }

        // Backtracking
        int i = lastIndex;

        // Until we reach an index which is its own parent
        while(parent[i] != i) {
            ans.push_back(nums[i]); // Add the element at current index
            i = parent[i]; 
        }
        ans.push_back(nums[i]); // Adding the last element 

        // Return the computed result
        return ans;
    }  
};

package main

import (
	"fmt"
	"sort"
)

// Function to find the largest divisible subset
func largestDivisibleSubset(nums []int) []int {
	n := len(nums) // Size of the array

	sort.Ints(nums) // Sort the numbers

	ans := []int{}              // To store the LDS
	dp := make([]int, n)        // DP array
	parent := make([]int, n)    // Array to keep record of the parent
	lastIndex := 0              // Index of last element in LDS
	maxLen := 0                 // Length of LDS

	// Initialize DP array to 1
	for i := 0; i < n; i++ {
		dp[i] = 1
	}

	// Computing the DP array
	for i := 0; i < n; i++ {
		parent[i] = i // Assign the parent to itself

		// For each previous index
		for prevInd := 0; prevInd < i; prevInd++ {

			// If the element at index i can be included in the LDS ending at index prevInd
			if nums[i]%nums[prevInd] == 0 && dp[i] < dp[prevInd]+1 {
				dp[i] = dp[prevInd] + 1 // Update the DP value
				parent[i] = prevInd     // Store the parent
			}
		}

		// If a longer LDS is found, update the values
		if dp[i] > maxLen {
			lastIndex = i
			maxLen = dp[i]
		}
	}

	// Backtracking
	i := lastIndex
	for parent[i] != i {
		ans = append(ans, nums[i]) // Add the element at current index
		i = parent[i]
	}
	ans = append(ans, nums[i]) // Adding the last element

	// Return the computed result
	return ans
}

func main() {
	nums := []int{3, 5, 10, 20}

	// Function call
	ans := largestDivisibleSubset(nums)

	// Output
	fmt.Print("The largest divisible subset is: ")
	for _, x := range ans {
		fmt.Print(x, " ")
	}
	fmt.Println()
}

Complexity Analysis

Time Complexity: O(N2), where N is the number of elements in the given array.
Sorting the array takes O(N logN). The nested for loops takes O(N2) and backtracking will take O(N) time in the worst case. Thus, the overall time complexity comes out to be: O(N2).

Space Complexity: O(N)
The DP array and parent array each takes O(N) space. In the worst case, the resultant array will store all the elements in the subset taking O(N) space. Thus, the overall space complexity is O(3N) or O(N).
