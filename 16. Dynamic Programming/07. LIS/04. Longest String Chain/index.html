Longest String Chain

Medium dsa

Problem Statement

You are given an array of words where each word consists of lowercase English letters.


wordA is a predecessor of wordB if and only if we can insert exactly one letter anywhere in wordA without changing the order of the other characters to make it equal to wordB.


For example, "abc" is a predecessor of "abac", while "cba" is not a predecessor of "bcad".


A word chain is a sequence of words [word1, word2, ..., wordk] with k >= 1, where word1 is a predecessor of word2, word2 is a predecessor of word3, and so on. A single word is trivially a word chain with k == 1.


Return the length of the longest possible word chain with words chosen from the given list of words.

Constraints

  • 1 <= words.length <= 1000
  • 1 <= words[i].length <= 20
  • words[i] only consists of lowercase English letters.

Example 1

Input: words = ["a", "ab", "abc", "abcd", "abcde"]

Output: 5

Explanation: The longest chain is ["a", "ab", "abc", "abcd", "abcde"].

Each word in the chain is formed by adding exactly one character to the previous word.

Example 2

Input: words = ["dog", "dogs", "dots", "dot", "d", "do"]

Output: 4

Explanation: The longest chain is ["d", "do", "dot", "dots"].

Each word is formed by inserting one character into the previous word.

Example 3

Input: words = ["a", "aa", "aaa", "aaaa", "b", "bb", "bbb"]

Intuition

The Intuition can be understood better from the following four points:

1. Extending the LIS idea
The problem of finding the longest string chain is conceptually similar to the Longest Increasing Subsequence (LIS) problem. In LIS, the goal is to find the longest sequence of numbers where each number is greater than the previous one. This establishes a natural order among elements. In the string chain problem, instead of numbers, the elements are words, and the order is defined by a transformation rule — specifically, one word can follow another if it can be formed by inserting exactly one character into the previous word.
2. Defining the Ordering Among Words
While LIS depends on a straightforward numeric comparison, the string chain problem requires a more complex relationship: each next word in the chain must be exactly one character longer and must contain all characters of the previous word plus one additional character. The problem establishes a chain based on the word length and the ability to derive the next word by insertion.
3. Leveraging Sorting and Dynamic Relationships
Sorting the array of words by their lengths enables a structured approach, allowing iteration from shorter to longer words. This ordering mimics the increasing order in LIS. For each word, the algorithm looks back to find possible predecessors that are exactly one character shorter and can be transformed into the current word by adding a single character. The longest chain ending at the current word is then the maximum chain length of any valid predecessor plus one.
4. Building the Longest Chain
By iterating through all words in ascending order of length and checking valid predecessors, the chain lengths can be computed incrementally. Maintaining the maximum chain length found so far gives the final answer.

Approach

  • Sort the words by their length in ascending order to ensure that when processing a word, all possible shorter predecessors have already been considered.
  • For each word, find all possible predecessors by removing one character at a time.
  • Check if the predecessor exists among the processed words.
  • Use dynamic programming to store and update the longest chain length ending at each word based on its valid predecessors.
  • Track the maximum chain length found during the iteration.

class Solution {
public:
    // Function to return the length of the longest string chain possible
    int longestStringChain(vector& words) {
        int n = words.size(); // Size of the array 
        sort(words.begin(), words.end(), compare);
        vector dp(n, 1); // DP array 

        // To store the length of longest string chain
        int maxLen = 0;

        // Computing the DP array 
        for(int i = 0; i < n; i++) {

            // For each previous index
            for(int j = 0; j < i; j++) {
                
                /* If the element at index i can be 
                included in the chain ending at index j */
                if(checkPossible(words[i], words[j]) && dp[i] < dp[j] + 1) {
                    dp[i] = dp[j] + 1; // Update the DP value
                }
            }

            // If a longer chain is found, update the values
            if(dp[i] > maxLen) maxLen = dp[i];
        }
        
        return maxLen;
    }
    
private:
    // Custom comparator function 
    bool static compare(string &s, string &t) {
        return s.size() < t.size();
    }
    
    // Function to check if the string can be added to the chain
    bool checkPossible(string &s, string &t) {
        //  Base case
        if(s.size() != t.size() + 1) return false;
        
        int i = 0, j = 0; // Pointers
        
        // Traverse until the first string is exhausted
        while(i < s.size()) {
            
            // Move both pointers if characters matches
            if(j < t.size() && s[i] == t[j]) {
                i++, j++;
            }
            // Otherwise 
            else {
                i++;
            }
        }
        
        // Return true if both the string gets exhausted
        if(i == s.size() && j == t.size()) return true;
        return false; // Return false otherwise
    }
};

import java.util.*;

class Solution {
    public int longestStringChain(String[] words) {
        int n = words.length; // Size of the array 
        Arrays.sort(words, Comparator.comparingInt(String::length));
        int[] dp = new int[n]; // DP array
        Arrays.fill(dp, 1);

        // To store the length of longest string chain
        int maxLen = 0;

        // Computing the DP array 
        for (int i = 0; i < n; i++) {

            // For each previous index
            for (int j = 0; j < i; j++) {

                /* If the element at index i can be 
                included in the chain ending at index j */
                if (checkPossible(words[i], words[j]) && dp[i] < dp[j] + 1) {
                    dp[i] = dp[j] + 1; // Update the DP value
                }
            }

            // If a longer chain is found, update the values
            if (dp[i] > maxLen) maxLen = dp[i];
        }

        return maxLen;
    }

    // Function to check if the string can be added to the chain
    private boolean checkPossible(String s, String t) {
        // Base case
        if (s.length() != t.length() + 1) return false;

        int i = 0, j = 0; // Pointers

        // Traverse until the first string is exhausted
        while (i < s.length()) {

            // Move both pointers if characters matches
            if (j < t.length() && s.charAt(i) == t.charAt(j)) {
                i++;
                j++;
            }
            // Otherwise 
            else {
                i++;
            }
        }

        // Return true if both the string gets exhausted
        return (i == s.length() && j == t.length());
    }
}

			}
		}

			maxLen = dp[i]
		}
	}

Complexity Analysis

Time Complexity: O(N2 × L), where N and L represents the number of words and the maximum length of any word respectively.
Sorting the words based on length takes O(NlogN) time. Computing the DP table takes O(N2) time. The checkPossible function compares both strings character by character, with a complexity proportional to the length of the longer string, which is at most O(L).

Space Complexity: O(N), due to the DP array used.

        return maxLen
    
    # Function to check if the string can be added to the chain
    def checkPossible(self, s, t):
        # Base case
        if len(s) != len(t) + 1:
            return False
        
        i, j = 0, 0  # Pointers
        
        # Traverse until the first string is exhausted
        while i < len(s):
            
            # Move both pointers if characters matches
            if j < len(t) and s[i] == t[j]:
                i += 1
                j += 1
            # Otherwise 
            else:
                i += 1
        
        # Return true if both the string gets exhausted
        if i == len(s) and j == len(t):
            return True
        return False  # Return false otherwise
