Number of Longest Increasing Subsequences

Medium dsa
ArrayBinary Indexed TreeDynamic ProgrammingSegment Tree

Problem Statement

Given an integer array nums, find the number of Longest Increasing Subsequences (LIS) in the array.

Constraints

  • 1 <= nums.length <= 103
  • -106 <= nums[i] <= 106

Example 1

Input: nums = [1, 3, 5, 4, 7]

Output: 2

Explanation: There are two LIS of length 4:

[1, 3, 4, 7]

[1, 3, 5, 7].

Example 2

Input: nums = [2, 2, 2, 2, 2]

Output: 5

Explanation: All elements are the same, so every single element can form an LIS of length 1. There are 5 such subsequences.

Example 3

Input: nums = [10, 9, 2, 5, 3, 7, 101, 18]

Editorial: Solution

Pre-requisites: Print LIS.

Intuition

To solve this problem, we can use the same tabulation approach as we used in printing the Longest Increasing Subsequence problem. We will maintain a dp array where dp[i] represents the length of the longest increasing subsequence that ends with the element at index i.

The only difference being that whenever we found a new way to update the dp array, we will also keep update the count of the longest increasing subsequence. We can do this by maintaining a count array where count[i] represents the number of longest increasing subsequences that end with the element at index i.

Approach

  • Initialize two arrays to track the length of the longest increasing subsequence ending at each index and the number of such sequences.
  • Traverse the array, and for each element, check all previous elements to build possible increasing subsequences.
  • Update the length and count for each position:
    • If a longer subsequence is found, update the length and reset the count.
    • If another subsequence of the same length is found, add to the count.
  • Track the maximum subsequence length throughout the traversal.
  • Sum up the counts of all subsequences that have the maximum length and return the result.

using namespace std;

class Solution {
public:
    /* Function to count the number of LIS 
    present in the given array */
    int numberOfLIS(vector nums) {
        int n = nums.size(); // Size of the array 

        int ans = 0; // To store the result
        int maxLen = 0; // To store the length of longest LIS

        vector dp(n, 1); // DP table
        vector count(n, 1); // To store the count

        // Filling up the DP table
        for(int ind = 0; ind < n; ind++) {
            for(int prevInd = 0; prevInd < ind; prevInd++) {

                /* If the element at index i can be
                included in the LIS ending at index j */
                if(nums[prevInd] < nums[ind]) {

                    // If a longer LIS is found, update the values
                    if(dp[prevInd] + 1 > dp[ind]) {
                        dp[ind] = dp[prevInd] + 1;
                        count[ind] = count[prevInd];
                    }

                    // Else if a new way is found to form the LIS
                    else if(dp[prevInd] + 1 == dp[ind]) {
                        count[ind] += count[prevInd];
                    }

                }
            }

            // Store the maximum length
            maxLen = max(maxLen, dp[ind]);
        }

        // Traverse the count array 
        for(int i = 0; i < n; i++) {
            // Count the longest LIS
            if(dp[i] == maxLen) ans += count[i];
        }

        return ans; // Return the result
    }
};

Time Complexity: O(N2), where N is the size of the given array.
Computing the DP array takes O(N2) time, and counting the longest LIS present takes O(N) time.

Space Complexity: O(N), because of two arrays (dp and count) used, each of size N.
