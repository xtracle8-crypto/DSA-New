Prime factorisation of a Number

Hard dsa
Math

Problem Statement

You are given an integer array queries of length n.

Return the prime factorization of each number in array queries in sorted order.

Constraints

  • 1 <= n <= 105
  • 2 <= queries[i] <= 2*105

Example 1

Input : queries = [2, 3, 4, 5, 6]

Output : [ [2], [3], [2, 2], [5], [2, 3] ]

Explanation : The values 2, 3, 5 are itself prime numbers.

The prime factorization of 4 will be --> 2 * 2.

The prime factorization of 6 will be --> 2 * 3.

Example 2

Input : queries = [7, 12, 18]

Output : [ [7], [2, 2, 3], [2, 3, 3] ]

Explanation : The value 7 itself is a prime number.

The prime factorization of 12 will be --> 2 * 2 * 3.

The prime factorization of 18 will be --> 2 * 3 * 3.

Example 3

Input : queries = [15, 20]

Editorial: Brute

Intuition:

A naive approach to find the prime factorization of a given number is by dividing the number multiple times with the smallest prime number that divides it until the number becomes 1.

The prime factorisation for a number can be found in the following manner:

  • The smallest prime number is 2. Thus, by repeatedly dividing the number by 2, all occurences of this prime factor are captured. This will reduce the number to an odd number.
  • Once the number is odd, the smallest possible factors start from 3. Instead of checking every number, all the even numbers can be skipped since they are not prime (except 2, which is already handled).
  • For each odd number up to the square root of the remaining number, if it is a factor, the number can be divided by this factorv epeatedly to capture all occurrences.
  • Edge Case: After processing with all possible factors up to the square root, if there is any number left greater than 2, it must be a prime factor. This is because any composite number would have been divided down to smaller factors already.

Approach:

  1. Prime Factorization of a Single Number:
    • A helper function is created to find the prime factors of a single number. It starts by repeatedly dividing the number by 2 to handle all factors of 2.
    • Then, it iterates through all odd numbers from 3 up to the square root of the number, dividing the number by these odd numbers to find and store their factors.
    • If the remaining number is greater than 2, it is also a prime factor and is added to the result.
  2. Processing the Queries:
    • The main function takes an array of integers and applies the helper function to each element in the array.
    • It stores the prime factors of each number in a 2D list where each sub-vector corresponds to the prime factors of a number in the input array.
  3. The computed result can be returned as the answer.

package main

import (
	"fmt"
	"math"
)

// Helper function to find the prime factorization of a number
func primeFact(n int) []int {
	// To store the result
	var ans []int

	// Divide by 2 while possible
	for n%2 == 0 {
		ans = append(ans, 2)
		n = n / 2
	}

	// Divide by odd numbers from 3 up to sqrt(n)
	for i := 3; i <= int(math.Sqrt(float64(n))); i += 2 {
		for n%i == 0 {
			ans = append(ans, i)
			n = n / i
		}
	}

	// Edge case: if n is a prime number > 2
	if n > 2 {
		ans = append(ans, n)
	}

	return ans
}

// Function to get prime factorizations for all numbers
func primeFactors(queries []int) [][]int {
	var ans [][]int

	for _, num := range queries {
		ans = append(ans, primeFact(num))
	}

	return ans
}

func main() {
	queries := []int{2, 3, 4, 5, 6}

	// Function call to get prime factorizations
	ans := primeFactors(queries)

	fmt.Println("The prime factorization of all the numbers is:")
	for i, factors := range ans {
		fmt.Printf("For %d: ", queries[i])
		for _, f := range factors {
			fmt.Printf("%d ", f)
		}
		fmt.Println()
	}

Complexity Analysis:

Time Complexity: O(N x num1/2) (where N represents the number of queries and num represents the average of all the numbers in queries)
The PrimeFact(num) function has the time complexity of O(num1/2):

  • The function divides the number by 2 until it is no longer divisible by 2. This loop runs at most log2num times.
  • The function then iterates over all odd numbers from 3 to sqrt(num), and for each odd number i, it keeps dividing num by i until num is no longer divisible by i. The number of iterations here depends on the number of prime factors of num.
  • In the worst case, the sum of the number of times each prime factor divides num is log2(num)
    (since 2 x 3 x 5 x 7 x ... up to (num)1/2 covers all prime numbers).
    • Therefore, the time complexity of the primeFact(num) function is O(num1/2).

The PrimeFactorials(N) function has the time complexity of O(N x num1/2) because for all N numbers, PrimeFact(num) is called.

Space Complexity: O(N x log2(num))
Using a list to store the prime factorization of a number takes up O(log2(num)) space. Thus, for N numbers, the total space requirement is O(N x log2(num)).

Editorial: Optimal

Intuition:

The optimal solution aims to efficiently find the prime factorization of each number in a given array. Unlike the naive approach that repeatedly divides by potential factors, this solution precomputes the smallest prime factor for every number up to a maximum value using the Sieve of Eratosthenes. This optimization significantly reduces the time complexity when querying prime factorizations for multiple numbers.

The use of the Sieve of Eratosthenes allows for the precomputation of prime factors, making the prime factorization query for each number extremely fast.

Approach:

  1. An array, SPF, is initialized where each index represents a number, and the value at that index represents the smallest prime factor of the number. Initially, all values are set to 1.
  2. Sieve of Eratosthenes:
    • Iterate through all numbers from 2 to MAX_N (100000 in this case).
    • For each prime number found (i.e., numbers whose SPF value is still 1), mark its multiples with the prime number as their smallest prime factor.
    • This process ensures that every number up to MAX_N has its smallest prime factor precomputed efficiently.
  3. Helper Function for Prime Factorization:
    • Given a number, repeatedly divide it by its smallest prime factor (obtained from the SPF array) until the number is reduced to 1.
    • Store each smallest prime factor during the division process.
  4. Handling Multiple Queries:
    • The main function takes an array of integers and applies the helper function to each element.
    • The prime factors of each number are stored in a 2D list, where each list corresponds to the prime factors of a number in the input array.

class Solution {
    private static final int MAX_N = 100000;

    /* To store the smallest prime factors for 
    all number with all nodes initially 
    having smallest prime factor as 1 */
    private int[] SPF = new int[MAX_N + 1];

    /* Constructor to initialize the smallest prime factor array */
    public Solution() {
        Arrays.fill(SPF, 1);
    }

    /* Function to precompute smallest prime 
    factor using sieve of eratosthenes */
    private void sieve() {
        // Iterate on all the number
        for (int i = 2; i <= MAX_N; i++) {
            // If the number is a prime number 
            if (SPF[i] == 1) {
                /* Mark all its multiples who have not 
                received their smallest prime factor yet*/
                for (int j = i; j <= MAX_N; j += i) {
                    // If smallest prime factor not received yet
                    if (SPF[j] == 1) {
                        /* Store i as the smallest 
                        prime factor for its multiple */
                        SPF[j] = i;
                    }
                }
            }
        }
    }

    /* Helper function to find the prime 
    factorization of a number */
    private List primeFact(int n) {
        // To store the result
        List ans = new ArrayList<>();

        // Until the number is not reduced to 1
        while (n != 1) {
            // Add the smallest prime factor to the list
            ans.add(SPF[n]);
            // Update the number
            n = n / SPF[n];
        }

        // Return the result
        return ans;
    }

    factorization of all the number */

        /* Precompute the smallest 
        possible factor for all numbers */
        sieve();

        List> ans = new ArrayList<>();

        // Iterate on each number in query
        for (int query : queries) {
            /* Function call to get the prime 
            factorization of current number */
            ans.add(primeFact(query));
        }

        // Return the answer
        return ans;
    }

Complexity Analysis:

Time Complexity: O(max_N x log(log(max_N))) + O(N x log(num)) (where N represent the number of queries, num represents the average number in the queries and max_N = 105)

  • The Sieve of Eratosthenes takes O(max_N x log(log(max_N))) to pre compute the smallest prime factor for all numbers till max_N.
  • In the worst case, finding the prime factors of a number num takes O(log(num)) time since the number is divided by its smallest prime factor until it becomes 1. For N queries, the total time taken will be O(N x log(num)).

Space Complexity: O(max_N) + O(N x log(num))
The SPF array takes O(max_N) space and the space taken by list to store the result is O(N x log(num)).

	// To store the result
	var ans []int

	// Divide by 2 while possible
	for n%2 == 0 {
		ans = append(ans, 2)
		n = n / 2
	}

	// Divide by odd numbers from 3 up to sqrt(n)
	for i := 3; i <= int(math.Sqrt(float64(n))); i += 2 {
		for n%i == 0 {
			ans = append(ans, i)
			n = n / i
		}

	// Edge case: if n is a prime number > 2

	}

	return ans
}
